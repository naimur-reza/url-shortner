# The Rules of Engagement

## 1. You Write Every Line
No AI-generated code for app logic. Setup/install/config is fine (that's boring). The real code? Your fingers, your brain.

## 2. The 10-Minute Rule
Stuck? Try for 10 minutes. After that, ask me:
- "Explain [concept]" — I teach
- "Debug this error" — I help find the bug
- "Is this approach okay?" — I review
- "Give me a hint" — I nudge, not tell

## 3. One Task Per Day
Each day has 1 task from the plan tracker. Finish it, check it off, feel good. Miss a day? Skip it, don't catch up. Consistency over intensity.

## 4. Ship Before Perfect
Working > beautiful. Ugly code that runs is better than perfect code that doesn't. You can refactor later.

## 5. Explain It Back
After writing something new, tell me in one sentence what it does. If you can't explain it, you didn't learn it.

## 6. No Copy-Paste Blindly
If you copy something from docs or me, you must read every line and understand it before pasting. Ask if unsure.

## 7. Review Every 3 Days
Every 3 sessions, re-read the code you wrote. Note 3 things you'd improve. That's how you level up.

## 8. Fun Rule — The Comeback
Every time you fix a bug yourself without help, you earn a "Comeback Point". Collect 3 and I owe you a joke or a meme on demand.

## 9. Don't Burn Out
If you're frustrated, say "break". We stop. No guilt. The code will be here tomorrow.

## 10. You're the Builder, I'm the Coach
I don't hold the keyboard. You do. My job is to point the way, call you out when you're slacking, and keep it fun.
